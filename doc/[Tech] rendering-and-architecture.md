# 웹 렌더링 기술 및 아키텍처 용어 정리

현대 웹 개발에서 프론트엔드 아키텍처와 렌더링 방식을 설명할 때 자주 등장하는 핵심 용어와 개념들을 정리한 문서입니다.

---

## 1. 아키텍처 패턴 (웹의 구조)

### MPA (Multi-Page Application, 다중 페이지 애플리케이션)
*   **개념:** 전통적인 웹사이트 방식. 사용자가 링크를 클릭할 때마다 서버에 새로운 페이지를 요청하고, 서버는 완성된 새 HTML 문서를 처음부터 끝까지 새로 만들어서 브라우저에 내려줍니다.
*   **장점:** 
    *   **완벽한 SEO (검색 엔진 최적화):** 완성된 HTML 문서가 제공되므로 크롤러 봇이 내용을 완벽하게 읽고 수집할 수 있습니다.
    *   **초기 로딩 (First Paint)이 빠름:** 브라우저가 무거운 자바스크립트를 다운받기 전에 화면에 내용(HTML)부터 먼저 보여줍니다.
*   **단점:** 화면 이동 시마다 브라우저가 처음부터 다시 화면을 그려야 하므로 하얗게 깜빡이는 현상(새로고침)이 발생하여 사용자 경험(UX)이 떨어집니다.

### SPA (Single Page Application, 단일 페이지 애플리케이션)
*   **개념:** 최초 한 번만 비어 있는 HTML 껍데기와 앱 전체를 구동하는 거대한 자바스크립트 파일 뭉치를 다운로드합니다. 그 후 사용자가 메뉴를 클릭하면 서버에 새 페이지를 요청하지 않고, 자바스크립트가 브라우저 내에서 알아서 화면의 일부분만 부드럽게 교체합니다. (React, Vue 등이 기본적으로 이 방식을 씁니다.)
*   **장점:** 
    *   **최고의 사용자 경험 (UX):** 화면이 깜빡이지 않고 모바일 앱처럼 부드럽고 즉각적으로 반응합니다.
    *   서버 부하 감소: 초기 로딩 이후에는 데이터(JSON 등)만 서버와 주고받으면 되므로 서버 자원을 아낄 수 있습니다.
*   **단점:** 
    *   **SEO에 매우 취약함:** 크롤러 봇이 빈 HTML 껍데기만 보고 "내용이 없다"고 판단하고 돌아갈 확률이 매우 높습니다.
    *   초기 로딩(JS 다운로드)이 무겁고 오래 걸릴 수 있습니다.

---

## 2. 렌더링 시점 (화면을 언제, 어디서 그리는가)

### CSR (Client-Side Rendering, 클라이언트 사이드 렌더링)
*   **개념:** 화면을 그리는 작업(렌더링)을 사용자의 브라우저(클라이언트)가 직접 수행하는 방식입니다. 주로 SPA에서 사용하는 방식입니다. 서버는 데이터만 주고, 브라우저가 자바스크립트를 이용해 HTML 태그를 조립해서 화면에 띄웁니다.

### SSR (Server-Side Rendering, 서버 사이드 렌더링)
*   **개념:** 사용자가 접속할 때마다 서버가 즉석에서 데이터베이스를 조회하고 HTML 문서를 완성하여(렌더링) 브라우저에 보내는 방식입니다. MPA 시절부터 쓰이던 고전적인 방식입니다. (항상 최신 데이터를 보여줘야 하는 뉴스나 게시판에 적합)

### SSG (Static Site Generation, 정적 사이트 생성)
*   **개념:** 코드를 작성하고 배포(Build)할 때 서버가 미리 모든 페이지의 완성된 HTML을 만들어 파일로 저장해 두는 방식입니다. 사용자가 접속하면 이미 만들어진 HTML 파일을 그냥 전달만 하므로 속도가 미친 듯이 빠릅니다. (내용이 자주 안 바뀌는 블로그, 소개 페이지, 현재 이 포트폴리오에 적합)

---

## 3. 현대적인 웹의 진화 (두 마리 토끼 잡기)

SPA(UX 최고, SEO 빵점)와 MPA(SEO 최고, UX 깜빡임)의 단점을 서로 보완하기 위해 등장한 기술들입니다.

### 유니버설 렌더링 (Universal Rendering) 또는 아이소모픽 앱 (Isomorphic App)
*   **개념:** 서버와 클라이언트(브라우저)에서 같은 코드가 동작할 수 있게 만드는 기술 구조입니다. (Next.js가 이 기술의 대표 주자입니다.)
*   **어떻게 두 마리 토끼를 잡는가?**
    1.  **첫 접속은 MPA처럼 (SSR 또는 SSG):** 크롤러 봇이나 사용자가 처음 주소로 치고 들어오면, 서버에서 미리 만들어둔 완성된 HTML을 던져줍니다. (SEO 완벽 해결, 빠른 초기 화면 표시)
    2.  **이후 작동은 SPA처럼 (CSR):** 화면이 뜨고 나면, 브라우저 뒤에서 자바스크립트가 조용히 깨어나 앱의 통제권을 넘겨받습니다(이 과정을 **Hydration**이라고 부름). 이제부터 사용자가 메뉴를 클릭하면 서버에 HTML을 다시 요청하지 않고, JS가 화면을 부드럽게 바꿔줍니다. (UX 완벽 해결)

### 프리패칭 (Prefetching)
*   **개념:** 사용자 몰래 미리 데이터를 가져오는 똑똑한 눈치 기술입니다.
*   **작동 방식:** 브라우저 화면에 '다음 페이지'로 가는 버튼이나 링크가 보이면, Next.js 같은 프레임워크는 "사용자가 곧 저길 누르겠군!" 하고 예상합니다. 그리고 사용자가 버튼에 마우스를 올리거나(hover) 화면에 보이는 순간, 백그라운드에서 다음 페이지에 필요한 데이터(JS, JSON 등)를 몰래 다운로드해서 캐싱해 둡니다.
*   **효과:** 사용자가 실제로 클릭하는 순간, 다운로드 대기 시간 0초! 이미 받아둔 데이터를 즉시 화면에 뿌려버리므로 화면 전환이 거의 빛의 속도로 이루어집니다. 이 기술 덕분에 URL이 바뀌는 유니버설 렌더링 방식에서도 기존 SPA 못지않은 즉각적인 반응성을 보여줄 수 있습니다.
