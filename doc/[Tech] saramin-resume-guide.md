# 사람인 이력서 작성 가이드

사람인(saramin) 이력서 편집을 도울 때 참고하는 지침.
각 섹션의 입력 필드 구조와 제약 조건을 정리한다.

---

## 1. MY Career (자기소개)

자유 텍스트로 본인을 소개하는 영역.

### 입력 필드

| 필드 | 유형 | 비고 |
|------|------|------|
| 자기소개 본문 | 텍스트 영역 | 자유 작성. 경력 요약, 강점, 지향점 등을 서술 |
| 핵심역량 | 태그 목록 | 최대 5개 (`ex1`~`ex5`). 짧은 키워드/문구로 핵심 역량을 표현 |
| URL 추가 | 링크 목록 | GitHub, LinkedIn, Brunch, Notion 등 외부 프로필 URL |

### 작성 팁

- 자기소개는 **구체적 성과**와 **기술 키워드**를 포함해야 채용 담당자의 눈에 띈다
- 핵심역량 태그는 지원 직무와 직접 관련된 역량으로 채운다
- URL은 실제 활동 내역이 있는 것만 등록한다 (빈 프로필 링크는 역효과)

---

## 2. 경력

재직했던 회사별로 경력 사항을 입력하는 영역.

### 입력 필드

| 필드 | 유형 | 필수 | 비고 |
|------|------|------|------|
| 회사명 | 텍스트 (검색) | O | 사람인 DB에서 검색하여 선택 |
| 입사년월 | YYYYMM | O | 예: `202308` |
| 퇴사년월 | YYYYMM | O | 재직중이면 비워두고 재직중 토글 활성화 |
| 재직중 | 토글 | - | 현재 재직 여부 |
| 직무 | 선택 | O | 드롭다운/검색으로 선택 (예: 앱개발) |
| 근무부서 | 텍스트 | - | 예: 소프트웨어개발부 |
| 직급 | 선택 | - | 선택하기 버튼으로 지정 (예: 사원, 대리 등) |
| 담당업무 | 텍스트 영역 | - | 자유 서술. 하단에 글자수/byte 표시 |
| 회사명 숨기기 | 체크박스 | - | 체크 시 회사명이 채용 담당자에게 비공개 |
| 퇴사사유 | 드롭다운 + 텍스트 | - | 선택형 또는 직접입력. 추가/제거 가능 |
| 연봉 | 추가 옵션 | - | `+ 연봉` 버튼으로 추가 |
| 근무지역 | 추가 옵션 | - | `+ 근무지역` 버튼으로 추가 |

### 작성 팁

- **담당업무**가 가장 중요한 필드. 단순 나열보다 **구체적 성과**와 **기술 스택**을 포함한 서술이 효과적
- 담당업무 예시 형식: `[프로젝트명] 기술스택을 활용하여 ~를 개발. 성과: ~`
- 퇴사사유는 부정적 표현을 최소화하되 솔직하게 작성

---

## 3. 학력

최종 학력 정보를 입력하는 영역.

### 입력 필드

| 필드 | 유형 | 필수 | 비고 |
|------|------|------|------|
| 최종학력 | 드롭다운 | O | 예: 대학·대학원 이상 졸업 |
| 학교유형 | 드롭다운 | O | 예: 대학교(4년) |
| 학교명 | 텍스트 (검색) | O | 사람인 DB에서 검색 |
| 편입 | 체크박스 | - | 편입 여부 |
| 전공 | 텍스트 | O | 예: 컴퓨터공학과 |
| 졸업상태 | 드롭다운 | - | 졸업 / 재학중 / 휴학중 / 중퇴 등 |
| 입학년월 | 날짜 | - | |
| 졸업년월 | 날짜 | - | |
| 주간/야간 | 드롭다운 | - | 주간 / 야간. 추가/제거 가능 |
| 학점 | 추가 옵션 | - | `+ 학점` 버튼으로 추가 |
| 추가전공 | 추가 옵션 | - | `+ 추가전공` 버튼으로 추가 (복수전공, 부전공 등) |
| 지역 | 추가 옵션 | - | `+ 지역` 버튼으로 추가 |
| 논문 & 졸업작품 | 추가 옵션 | - | `+ 논문 & 졸업작품` 버튼으로 추가 |

### 작성 팁

- 학점이 좋다면 반드시 추가한다 (3.5/4.5 이상 권장)
- 졸업작품이나 논문이 지원 직무와 관련 있다면 추가하면 좋다

---

## 4. 스킬

보유 기술/역량을 태그 형태로 등록하는 영역.

### 입력 필드

| 필드 | 유형 | 비고 |
|------|------|------|
| 스킬 검색 | 텍스트 (검색) | 툴/직무역량/소프트스킬을 검색하여 추가 |
| 나의 스킬 | 태그 목록 | 추가된 스킬이 태그로 표시. 각 태그에 편집/삭제 가능 |
| 소프트스킬 불러오기 | 버튼 | 인성검사 결과 기반 소프트스킬 추가 |

### 작성 팁

- 지원 직무의 채용공고에서 자주 언급되는 기술을 우선 등록한다
- 하드스킬(프로그래밍 언어, 프레임워크, 도구)과 소프트스킬을 균형 있게 배치
- 실무에서 사용한 경험이 있는 스킬만 등록한다 (면접에서 검증될 수 있음)
- 스킬 수는 6~10개 정도가 적당. 너무 많으면 전문성이 희석되어 보인다

---

## 문서 구조

이력서 마크다운 파일(`doc/[Career] saramin-resume.md`)은 아래 구조를 따른다:

```markdown
# 사람인 이력서

## MY Career
(자기소개 본문)

## 경력
### 회사명
- **기간**: YYYY.MM ~ YYYY.MM (N년 N개월)
- **부서**: ...
- **직급**: ...
- **업무**: ...
- **퇴사사유**: ...

## 학력
- **최종학력**: ...
- **학교**: ...
- **전공**: ...

## 스킬
스킬1, 스킬2, ...
```

---

## 에이전트 작업 시 주의사항

1. **마크다운 파일은 사람인 웹사이트의 입력 필드와 1:1 대응**된다. 마크다운에서 수정한 내용을 사용자가 사람인 웹에 직접 복사-붙여넣기 할 수 있도록 형식을 맞춘다
2. **담당업무** 텍스트 영역의 글자수 제한에 유의한다 (사람인 UI에 글자수/byte 카운터가 표시됨)
3. 이력서 내용을 보완할 때는 사용자의 실제 경험과 역량에 기반하여 작성한다. 허위 사실을 추가하지 않는다
4. 수정 제안 시 **변경 전/후**를 명확히 보여주어 사용자가 검토할 수 있게 한다
