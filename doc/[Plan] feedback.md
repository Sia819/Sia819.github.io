## 문체/톤 개선 (중규모)

- [ ] introduction.md 전면 재작성
  - **문제**: "~입니다", "~습니다" 자소서 투 반복. "깔끔한 구조와 유지보수 가능한 코드를 지향합니다" 같은 추상적 클리셰
  - **변경 포인트**:
    1. 3~4개 짧은 문단으로 분리 (현재 3개인데 각각이 긺)
    2. 추상적 → 구체적 경험으로 교체
       - "혼자서도, 팀에서도 돌아가는 개발" → 구체적 상황 제시 (예: "3인 팀에서 백엔드를 단독 설계·운영하며, 클라이언트 구조 설계까지 병행")
       - "왜 이 구조인지 설명할 수 있는 코드" → 빼거나, "MVVM 패턴 도입으로 UI-로직 분리를 표준화" 같은 실례로 교체
       - "필요하면 직접 찾아서 해결하는 자세" → "GPU 서버 구축부터 사내 AI 챗봇 운영까지, 개발 외 인프라도 직접 담당" 같은 구체 사례
    3. 동사 다양화: "주도했습니다" 반복 줄이기

- [ ] career.md 서술 다듬기
  - **문제**: "~ 구현", "~ 개발", "~ 처리"가 줄줄이 반복. 기술 나열에 치중하고 "왜/어떤 문제를 풀었는가"가 부족
  - **변경 포인트**:
    1. 각 프로젝트 도입부에 **한 줄 임팩트** 추가 — "이 프로젝트로 무엇이 달성되었는가"
       - 예) 디지털 트윈: "제조공장 설비를 3D 환경에서 시뮬레이션하여 최적 배치를 도출하는 솔루션을 설계부터 납품까지 리드"
    2. 동사 다양화:
       - "구현" → 설계, 도입, 적용, 연동, 자동화, 표준화, 최적화
       - "개발" → 구축, 설계, 제작
    3. 기술적 챌린지 언급 추가 (가능한 곳만):
       - 예) "6차 다항식 계수의 부동소수점 정밀도 문제를 발견하고 해결" — 이미 있지만 더 구체적으로
       - 예) "FBX → glTF 변환 파이프라인에서 서버단 Blender 연동으로 런타임 로드 성능 확보"

- [ ] resume.md 기술 스택 설명 보강
  - **문제**: backtick 나열만으로 정보 밀도가 낮음
  - **변경**: 각 카테고리에 한 줄 설명 추가
    - Core: "실무 프로젝트에서 설계부터 운영까지 주도한 기술"

    - → 현재도 "실무 주력", "프로젝트 투입 경험"이 있으므로, 이 부분은 미세 조정 수준
